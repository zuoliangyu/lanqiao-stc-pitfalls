### （1）数码管卡死,显示3个5无法显示温度数据

* **问题描述**：
* **问题背景**：哔哩哔哩课程零基础部分，采用蓝桥杯单片机仿真
* **问题分析**：

  + 数码管底层中,段码位码没有设置成全局变量
  + 数码管减速变量设置为了unsigned char 而不是unsigned int ,减速值又设置为500
  + isp生成定时器选择了自动重装载,但中断服务函数又额外加上定时器初始值
* **解决方案**：

  + 合理配置定时器,修改Seg底层

```
// 关键错误代码示例
/*******************************************定时器0初始化****************************************/
void Timer0Init(void)		//1毫秒@12.000MHz
{
	AUXR &= 0x7F;		//定时器时钟12T模式
	TMOD &= 0xF0;		//设置定时器模式
	TL0 = 0x18;		//设置定时初值
	TH0 = 0xFC;		//设置定时初值
	TF0 = 0;		//清除TF0标志
	TR0 = 1;		//定时器0开始计时
	EA=1;
	ET0=1;
}

/*******************************************定时器0服务函数****************************************/
void fuwu() interrupt 1
{
	TL0 = 0x18;		//设置定时初值  !!!!错误部分
	TH0 = 0xFC;		//设置定时初值 !!!!错误部分
	if(++key_slow==10)key_slow=0;
	if(++seg_slow==500)seg_slow=0; //!!!!错误部分
	if(++led_pose==8)led_pose=0;
	led_disp(led_pose,ucled[led_pose]);	
	if(++seg_pose==8)seg_pose=0;
	seg_disp(seg_pose,seg_buf[seg_pose],seg_point[seg_pose]);
}
```

### （2）USART通信数据错误

* **问题描述**：短按数据一直跳,长按正常。
* **问题背景**：哔哩哔哩零基础课程，按键长短按部分。
* **问题分析**：

  + 短按逻辑错误,没有保证短按只触发一次,而是随按键检测重复触发
* **解决方案**：

*在有长按的情况下,修改为正确的短按逻辑。  
*修改为视频中正确的短按模板
