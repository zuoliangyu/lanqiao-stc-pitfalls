## 三、本周重点问题详解
### （1）变量类型错误（多届重复出现）
- **问题描述：** DAC电压显示错误 / LED位置错位 / N值显示乱码 / 串口收不到回答
- **问题背景：** 第12届（DAC值）、第15届国赛（freq截断、sscanf越界）、第16届（ucLed、sprintf）
- **问题分析：** 变量类型选择错误或缺失导致数据溢出、截断、内存越界：
```c
// 第12届：Da_Volt_100x 为 unsigned char，存不下100倍电压
unsigned char Da_Volt_100x; // 最大255，存不下355、175等值
Da_Volt_100x = 500; // 溢出为244
// 第15届国赛：freq 为 unsigned char，频率超255截断
unsigned char freq;
freq = (TH0 << 8) | TL0; // 1000Hz → 1000 & 0xFF = 232
// 第16届：ucLed 缺 char，unsigned 默认 unsigned int(2字节)
pdata unsigned ucLed[8]; // 2字节/元素，Led_Disp按1字节读取，位置全错
// 第16届：sprintf 中 unsigned char 传入 %d，Keil C51压栈1字节但%d读2字节
sprintf(seg_string, "N%1d", light_level); // light_level=4 → 显示N1
// 第15届国赛：sscanf %u 向 unsigned char* 写2字节，踩踏相邻变量
unsigned char x, y, result;
sscanf(buf, "(%u,%u)", &x, &y); // result被踩坏，后续判断失败
```
- **解决方案：** 根据数据范围选择合适类型；Keil C51中 `sprintf`/`sscanf` 的参数必须显式转为 `(unsigned int)` 或使用 `unsigned int` 变量
> **规律总结：** 本周出现频率最高的Bug类型（6次）。核心原则：凡是参与乘法运算（×100、×51）的变量用 `unsigned int`；`sprintf`/`sscanf` 的参数在 Keil C51 中必须与格式符宽度匹配；声明数组时 `unsigned` 不等于 `unsigned char`，必须写全。
### （2）LED/显示逻辑错误（多届重复出现）
- **问题描述：** LED始终全灭 / 存在错误指示灯 / L1指示条件写错 / L2/L3永远不熄灭
- **问题背景：** 第12届（先显后更新）、第15届（L2误亮、L1条件错）、第16届（缺Led_Disp调用、Led_Off干扰闪烁）
- **问题分析：**
```c
// 第12届：先显示再更新，LED状态滞后一帧
void led_proc() {
led_disp(ucled); // ← 先显示旧值
ucled[0] = (set_mode == 0); // ← 再更新
}
// 第16届：设置了ucLed数组但从未写入硬件
void Led_proc() {
Led_Off();
ucled[0] = ...;
// ❌ 缺少 Led_Disp(ucLed); ← 硬件永远不更新
}
// 第16届：Led_Off每1ms执行，LED硬件每1ms经历灭→亮跳变
// 测试设备检测到的是1ms闪烁而非预期的100ms闪烁
// 第15届：L1判断用显示页面而非DAC模式
if(Seg_mode == 0) ucLed[0] = 1; // ❌ 应该判断Dac_mode
```
- **解决方案：** 先更新数组再调用显示函数；确保 `Led_Disp()` 被调用；删除不必要的 `Led_Off()`；L1指示条件改为 `Dac_mode==0`
> **规律总结：** LED相关Bug的共性是"数据与硬件脱节"——要么更新了数组没写硬件，要么写硬件的时机/条件不对。检查要点：(1)有没有调用Led_Disp；(2)调用顺序是否正确；(3)指示条件是否对应题意。
### （3）初始化遗漏（多届出现）
- **问题描述：** 系统完全不工作 / 上电后DAC比较值为0 / 开机LED误亮 / 首次测距假跳变
- **问题背景：** 第15届国赛（ET1=0、Timer0Init未调用）、第12届（Temperature_Ind未初始化）、第15届省赛（T_float未初始化）、第16届（distance初始为0）
- **问题分析：**
```c
// 第15届国赛：Timer1Init中写成ET1=0; EA=0; 系统完全瘫痪
ET1 = 0; // ❌ 定时器中断未开，uwtick永远为0，调度器不运行
EA = 0; // ❌ 全局中断关闭，覆盖了UartInit中的EA=1
// 第12届：Temperature_Ind未初始化，上电默认0而非题目要求的25
idata unsigned char Temperature_Ind; // = 0，不是25
// DAC判断 18 < 0 → FALSE → 输出5V（应该18 < 25 → TRUE → 输出0V）
// 第15届省赛：T_float初始为0，首次温度23°C触发"温度上升"标志
// T_float(0) < temp(23) → T_up_flag = 1 → L2误亮
// 第16届：distance初始为0，首次测距|16-0|=16≥10，误判为"跑动"
```
- **解决方案：** 检查所有变量初始值是否符合题目要求的"初始状态"；确保所有Init函数都被调用；注意Init函数的调用顺序（后调用的会覆盖先调用的设置）
> **规律总结：** 初始化遗漏往往在"上电后第一次测试"就暴露。赛前检查清单：(1)题目要求的初始参数值是否都赋了初值；(2)所有Init函数是否都调用了；(3)Init顺序是否会互相覆盖（如EA=1被后续EA=0覆盖）。
### （4）Relay()函数逻辑错误
- **问题描述：** 所有继电器测试失败（测试41/42/44/45/46全错）
- **问题背景：** 第15届省赛
- **问题分析：** `Relay()` 函数直接修改了 P0，但从未修改状态缓存变量 `temp_2`：
```c
void Relay(bit enable) {
if(enable)
P0 |= 0x10; // 直接改P0，temp_2还是0x00
else
P0 &= ~0x10;
// temp_2未变 → temp_2 == temp_2_old → 下次不触发锁存器
// 即使第一次触发，P0 = temp_2 也会用0x00覆盖掉继电器位
}
```
- **解决方案：** 改为修改 `temp_2`，再通过锁存器输出：
```c
void Relay(bit enable) {
if(enable) temp_2 |= 0x10;
else temp_2 &= ~0x10;
if(temp_2 != temp_2_old) {
P0 = temp_2;
// 锁存到Y5(0xA0)
temp_2_old = temp_2;
}
}
```
> **规律总结：** 蓝桥杯开发板的LED、继电器、蜂鸣器共用P0口通过138译码器锁存。操作这些外设时必须通过缓存变量（`temp_1`/`temp_2`）间接操作，不能直接写P0，否则不同外设之间会互相覆盖。
### （5）按键作用域与继电器锁定逻辑
- **问题描述：** 模式切换无效 / 锁定状态下继电器仍被操作 / S17继电器方向写反
- **问题背景：** 第12届（S5放在case内）、第15届（锁定未屏蔽、方向写反）
- **问题分析：**
```c
// 第12届：S5模式切换只在DAC界面case 2内有效
case 2: // DAC输出界面
if(key_down == 5) set_mode ^= 1; // ← 只有在DAC页面才能按！
break;
// 题目要求S5是全局模式切换，任何界面都能按
// 第15届：锁定状态下else分支仍操作继电器
if(suoding == 0)
Relay(1);
else
Relay(0); // ❌ 锁定时应什么都不做，不是关闭继电器
// 第15届：S17应关闭继电器，写成了Relay(1)打开
```
- **解决方案：** 全局按键处理移到 `switch` 外面；锁定时不进入继电器操作分支；仔细核对S16(开)/S17(关)与Relay(1)/Relay(0)的对应关系
> **规律总结：** 按键处理放置位置决定了它的作用域。全局功能键（模式切换等）不能放在 `switch(显示页面)` 的某个 `case` 里。继电器/锁定逻辑要画出状态表再写代码，避免方向写反或锁定状态遗漏。

## 五、经验教训与注意事项
1. **数据类型选择：** 涉及乘法运算（×100、×51）或存储范围超过255的变量必须用 `unsigned int`；Keil C51中 `sprintf`/`sscanf` 参数必须显式转为 `(unsigned int)` 或直接声明为 `unsigned int`；`unsigned` 不等于 `unsigned char`，必须写全 `unsigned char`
2. **LED三检法：** (1)有没有调用 `Led_Disp()`？(2)是先更新数组还是先调用显示？(3)指示条件是否对应题意（`Dac_mode` vs `Seg_mode`）？额外注意 `Led_Off()` 不要在每次循环中调用，会导致闪烁间隔异常
3. **初始化检查清单：** (1)题目要求的初始参数值是否赋了初值？(2)所有Init函数是否都调用了？(3)Init顺序是否会互相覆盖？(4)传感器首次读数前是否需要预读/预等待？
4. **继电器/蜂鸣器操作：** 必须通过 `temp_2` 缓存变量操作，不能直接写P0。修改后通过138译码器Y5(0xA0)锁存
5. **按键作用域：** 全局功能键不能放在 `switch(页面)` 的 `case` 内部；继电器锁定时应跳过操作而非执行关闭
