### (1)Proteus仿真报错

* **问题描述**:启动Proteus仿真时出现报错,无法正常运行仿真。
* **问题背景**:无具体课程背景。
* **问题分析**:

  + Proteus软件需要管理员权限才能正常访问某些系统资源
  + 权限不足导致仿真无法启动或报错
  + 这是Windows系统权限管理导致的常见问题
* **解决方案**:

  + 以管理员身份启动Proteus软件。右键点击Proteus快捷方式,选择”以管理员身份运行”,即可解决权限问题。

### (3)IIC代码与大模板有差异

* **问题描述**:官方给的底层IIC代码和大模板的有差异,不确定差异部分是否需要补上。
* **问题背景**:无具体课程背景。

### (4)数码管不亮

* **问题描述**:数码管无任何显示,完全不亮。
* **问题背景**:无具体课程背景。
* **问题分析**:

  + 中断服务函数定义不完整
  + 缺少中断号导致中断向量表无法正确映射
  + 中断函数无法被正确调用
* **解决方案**:

  + 中断服务函数没有加中断号。需要在中断服务函数定义时添加中断号,例如:

```
// 错误示例
void Timer0_ISR() interrupt  // 缺少中断号
{
    // 中断处理代码
}

// 正确示例
void Timer0_ISR() interrupt 1  // 添加中断号1(Timer0)
{
    // 中断处理代码
}
```

### (5)LED1亮灭不稳定

* **问题描述**:LED1亮灭状态不稳定,切换异常。
* **问题背景**:无具体课程背景。
* **问题分析**:

  + 连续读取光敏和电位器的数值时,ADC通道切换不当
  + 会导致数据互换或采样错误
  + 判断逻辑基于错误的数据,导致LED状态不稳定
* **解决方案**:

  + 读取完一个传感器数据后,立即进行判断和处理,再读取下一个传感器数据。避免连续读取多个ADC通道导致数据混乱。每次读取后要有足够的时间间隔,确保ADC转换完成。

---

### (6)超声波数据存储时间相同

* **问题描述**:超声波数据还不能存储到数组里,显示的时间是一样的,每次存储的时间戳都相同。
* **问题背景**:十七届4T场次一模拟赛。
* **问题分析**:

  + 更新RTC时间的函数放在了一个特定界面的处理函数里
  + 只有在特定界面时才会更新时间
  + 在其他界面存储数据时,时间没有更新,所以每次存的时间都是一样的
* **解决方案**:

  + 将更新RTC时间的函数放到主循环或系统调度函数的开头,确保每次循环都执行到时间更新函数,这样存储数据时才能获取到实时更新的时间戳。

### (7)串口中断TI标志处理疑问

* **问题描述**:串口中断为什么不用管TI标志?当我发送字节后TI=1跳入串口中断,没对TI进行操作,出来后仍然是1又跳进去,不会一直卡在中断吗?
* **问题背景**:无具体课程背景。
* **问题分析**:

  + 学员对串口中断标志位的处理流程理解不清
  + 担心TI标志不清零会导致重复触发中断
  + 不了解printf重定义函数的内部实现
* **解决方案**:

  + 在printf重定义函数里,串口发送完毕后,会把TI标志再赋值成0,所以不会一直卡在中断。
