### （1）计时变量溢出

* **问题描述**：程序中定时器初始化正确，中断正确，数码管显示出问题上图;LED闪烁出问题，下图

* **问题背景**：采用单片机仿真图观察现象
* **问题分析**：

  + 检查定时器配置
  + 检查中断号与是否手动重装载
  + 计时变量1ms自增，类型定义为unsigned char，
* **解决方案**：

* 将计时变量unsigned char类型改为unsigned int
* 提醒用户注意变量定义

### （2）按键卡死

* **问题描述**：按下按键无反应

* **问题背景**：\
* **问题分析**：

  + 检查按键底层，发现temp未赋初始值
  + 检查主函数Key_Proc函数
  + 检查控制逻辑
* **解决方案**：

  + 底层temp赋初始值0
  + 控制逻辑修改：你这个按键，你按一下他，他会执行一次，你这个Index变量放在按键里面加加，按一次她才变化一次。所以说你要实现自动流转的过程，然后在那个逻辑里面，不是有那个switch选择流转模式嘛，你在每一种的结尾加判断条件，让他平滑的过渡到下一个模式。

### （5） 数码管显示数据不正确

* **问题描述**：用户在长按跳转界面显示模式下的数据错误
* **问题背景**：过渡训练–电压采集器
* **问题分析**：\

  + 检查长按逻辑
  + 查找显示数据处理，发现显示数据处理不正确，LED_Mode与Seg_Mode混淆用错，导致数据错误
* **解决方案**：

  + 将Seg_Mode改成LED_Mode即可在对应LED展示模式下采集数据

### （6） 电压采集代码为啥放在Seg_Proc前面？

* **问题描述**：电压采集代码为啥放在Seg_Proc前面？而不放在Key_Proc中

* **问题背景**：\
* **问题分析**：\

* **解决方案**：

  + 放到数码管前面，那每次显示的数据就是更新的，放在按键那边，他可能数据没更新啊。其实也可以放在按键那边，一般这种关于数据采集，都放在数码管显示前面比较好，从代码的规范性来说。

### （8） 按键底层无法理解

* **问题描述**：按键底层，if(P3_4 == 0) temp=0等代码不理解，主函数Key_Proc按键解析代码不理解

* **问题背景**：\
* **问题分析**：\
* **解决方案**：通过原理图逐步解析按键，按下时P3_4 == 0说明按键1按下了，返回此时键码值1。那么同理，按键2按下就返回2。一定要赋初始值temp=0，当没有按键按下时无初始值键码会为随机值。Key_UP==2就是按键2抬起来了，松手了。
