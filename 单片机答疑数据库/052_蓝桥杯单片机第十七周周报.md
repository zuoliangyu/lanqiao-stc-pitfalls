## 三、本周重点问题详解
### （1）PWM 频率平滑切换逻辑问题
- **问题描述**：学生在实现 PWM 频率从 4000Hz 逐步增加到 8000Hz（或反向）的平滑过渡时，频率变化不符合预期。
- **问题背景**：采用 STM32G431RBT6，使用 HAL 库，利用 `uwTick` 进行非阻塞延时处理。
- **问题分析**：
- 步进逻辑判断有误：在 100ms 周期内，频率增加量与目标频率的判定条件（`data.pwm_freq != (pwm_mode?8000:4000)`）导致逻辑死循环或越界。
- 时间轴计算不清晰：从 4000Hz 到 8000Hz，步进为 80Hz，需 50 次调整。
- **解决方案**：
- **逻辑修正**：将==改为>=就可解决，因为等于属于严格判断，必须恰好相等才可。而大于等于为范围判
断，满足大于或等于即可。明确每次步进 80Hz，配合 100ms 的判断周期，确保整个切换过程在 5 秒内平稳完成。
- **代码参考**：
```
if(uwTick - now_time2 >= 100)
{
now_time2 = uwTick;
if(data.pwm_freq != (pwm_mode?8000:4000))
{
data.pwm_freq += 80 * (pwm_mode?1:-1);
pwm_set_frequency(data.pwm_freq);
}
}
```

### （2）LED 锁存器控制与引脚复用问题
- **问题描述**：LED 初始化后状态不正确，且容易受到程序其他部分（如 LCD 刷新）的影响。
- **问题背景**：蓝桥杯嵌入式开发板，LCD 与 LED 共用 `PC8` - `PC15` 数据引脚。
- **问题分析**：
- **硬件互锁**：LED 是通过锁存器（74LS573）控制的，必须通过 `PD2`（LE 锁存端）的高电平触发来更新状态。
- **资源冲突**：由于 LCD 与 LED 复用 PC 口引脚，若不及时通过 `PD2` 锁定 LED 状态，LCD 的信号会直接导致 LED 闪烁或误点亮。
- **解决方案**：
- **规范流程**：编写 LED 初始化及控制函数时，需遵循“送数据到 PC 口->拉高 PD2->拉低 PD2”的步骤来锁定电平。
- **电路特性**：明确该板卡 LED 为低电平点亮特性。
