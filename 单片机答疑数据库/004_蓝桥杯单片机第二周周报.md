### （1）数码管显示错误

* **问题描述**：数码管显示情况与`seg_buf[]`数组中给的初始值不一致。
* **问题背景**：[b站] [零基础入门三] [数码管]，采用proteus仿真图。
* **问题分析**：

  + 学员想显示123456 却显示了124567 首先检查数组长度是否和0-9 10灭相符合
* **解决方案**：

  + 学员段码少写一位导致显示错位，示例代码修改如下：

```
uint8 seg_dula[]={0x3F,0x06,0x5B,0x4f,0x66,0x6d,0x7d,0x07,0x7f,0x6f,0x00};
/*  学员段码少写一位导致显示错位 以下为原先错误数组
uint8 seg_dula[]={0x3F,0x06,0x5B,0x66,0x6d,0x7d,0x07,0x7f,0x6f,0x00};
*/
uint8 seg_wela[]={0xfe,0xfd,0xfa,0xF7,0xef,0xdf};
void seg_disp(uint8 wela,dula)
{
	P0=0x00;
	P26=1;P26=0;
    
	P0=seg_wela[wela];
	P27=1;P27=0;
	
	P0=seg_dula[dula];
	P26=1;P26=0;
}
```

### （2）按键失灵

* **问题描述**：模块化完后编译一直报错。
* **问题背景**：[b站] [蓝桥杯大模板] [按键]，采用proteus仿真图。
* **问题分析**：

  + 是不是按键初始未赋值
  + 是不是按键初始值定义的不是局部变量及按键未按下时`uint8 key_read(void)`返回值应该为0
* **解决方案**：

  + 检查学员底层，发现按键返回值给的全局变量，告诉学员局部变量，全局变量以及`static uint8 与 uint8`在局部的区别，做好了区分

| 变量类型 | 作用领域 | 存在周期 |
| --- | --- | --- |
| 全局变量 | 从定义，到结束；跨文件使用加个extern | 程序整个运行期间 |
| 局部变量 | 函数 / 代码块内部 | 函数调用期间 |
| 静态局部变量 | 函数 / 代码块内部（与局部变量相同） | 程序整个运行期间（与全局变量相同） |

### （5）延时时间对不上

* **问题描述**：流水灯延时时间很奇怪。
* **问题背景**：[b站] [决赛试题与过渡模拟] [延时] ，采用proteus仿真图。
* **问题分析**：

  + 检查Delay底层 看看`unsigned int xms`是否写成了`unsigned char xms`
* **解决方案**：

  + 将`xms`变量类型扩大，延时时间恢复正确

```
/*
学员原先是 void Delay(unsigned char xms)
这样在 xms在255以上时会自动转为unsigned char 比如传入500会转成500-255=245的延时 导致对不上
*/
void Delay(unsigned int xms)
{
	unsigned char data i, j;
	while(xms--)
    {
        i = 2;
        j = 239;
        do
        {
            while (--j);
        } while (--i);
    }
}
```

### （7）不知道如何通过中断控制LED流水

* **问题描述**：只会用按键按一下灯向左向右流水一次，无法选择方向进行流水
* **问题背景**：[b站] [决赛试题与过渡模拟] [逻辑]，采用proteus仿真图。
* **问题分析**：

  + 首先可以根据按键选择模式 比如：按键一按下选择模式一，以此类推
  + 流水可以开个`time_500ms`变量在定时器中断不断加加直到500执行函数
  + 执行函数可以根据mode选择左移还是右移
* **解决方案**：

  + 两个灯流水这种类似的交给学员自己写，这边提供了一个流水方式

```
void key_proc()
{
	if(key_slow) return;
	key_slow=1;
	key_val=Key_Read();
	key_down=key_val&(key_val^key_old);
	key_up=~key_val&(key_val^key_old);
	key_old=key_val;
	/* 开始 */
    //按下1切换成模式1以此类推
	if(key_down)
		mode=key_down;
}

//定时器代替delay
void Timer0Server() interrupt 1
{
	TL0 = 0x18;
	TH0 = 0xFC;
	if(++key_slow ==10 ) key_slow=0;
	/* 流水灯相关 */
	if(++time_500ms==500) 
	{
		time_500ms=0;
        if(mode==0)return;//初始什么都不点亮也不流水
        if(P1=0xFF)
        	P1=0xFE;
        else 
        {
            switch(mode)
            {
                case 1://右移动
                    P1=_cror_(P1,1);
                break;

                case 2://左移动
                    P1=_crol_(P1,1);
                break;
            }
        }
    }
}
```

### （8）彩灯系统的底层如何写？

* **问题描述**：感觉用常规底层解决各种模式切换及其麻烦
* **问题背景**：[b站] [决赛试题与过渡模拟] [逻辑]，采用proteus仿真图。
* **问题分析**：

  + 我们可以抛弃底层，创建两个数组，一个单个亮，一个成对亮
  + 然后开两个索引，分别对应两个数组的所处位置
  + 进行流水模式选择
* **解决方案**：

  + 我们采用伪代码提供思路

```
  unsigned char led_single[8]={第一个灯亮，...,第八个灯亮};
  unsigned char led_couple[4]={第一个和第八个灯亮，...，第四个和第五个灯亮};
  unsigned char single_index;
  unsigned char couple_index;
  
  void key_proc()
  {
  	if(key_slow) return;
  	key_slow=1;
  	key_val=Key_Read();
  	key_down=key_val&(key_val^key_old);
  	key_up=~key_val&(key_val^key_old);
  	key_old=key_val;
  	/* 开始 */
      //按键三 模式加一 
      //按键四 模式减一
  	switch(key_down)
      {
  		case 3:
              mode++;
              if(mode==5)mode==1
  		break;
  
  		case 4:
               mode--;
              if(mode==0)mode=4;
  		break;
      }

  void Timer0Server() interrupt 1
  {
  	TL0 = 0x18;
  	TH0 = 0xFC;

  	/* 流水灯相关 */

  	{
  		time_500ms=0;
  		switch(mode)
  		{
  			case 1:// 1 ——> 2 ——> 3 ——> 4 ——> 5 ——> 6 ——> 7 ——> 8
                  P1=led_single[single_index++];
                   if(single_index==8)single_index=0;
               break;
  
               case 2:// 8 ——> 7 ——> 6 ——> 5 ——> 4 ——> 3 ——> 2 ——> 1
                   P1=led_single[single_index--];
                   if(single_index==8)single_index=0;
  			break;
  
               case 3:// 1-8 ——> 2-7 ——> 3-6 ——> 4-5
  				P1=led_couple[couple_index++];
                  if(couple_index==4)couple_index=0;
               break;
  
               case 4:// 4-5 ——> 3-6 ——> 2-7 ——> 1-8
```

### （9）c语言冒泡排序

* **问题描述**：无法实现排序，但已经通过ai修正
* **问题背景**：[b站] [may] [C语言]。
* **问题分析**：

  + 编译代码发现可以运行成功
* **解决方案**：

  + 从学员的操作来看 他的`scanf`用`,`分割每一个输入的数字1,12,3,45,6,7,8,9,12,32。导致结果只录入了一个数字
  + 关于`scanf`的录入我们可以用空格，或者`Enter`隔开达到输入的完整性

```
#include<stdio.h>
int a[10],i,j,x,temp;
int main()
{
	for(i=0;i<10;i++)
    {
		scanf("%d",&a[i]);
		if((a[i]>10000)||(a[i]<0)){
			printf("输入的数据有误\n");
			return 1;
		}
	}
	
	for(j=0;j<=8;j++)
	{
		x=j;
        for(i=j+1;i<=9;i++)
		{
			if(a[i]>a[x])
			{
				x=i;
			}
		}
		if(x!=j)
         {
			temp=a[j];
			a[j]=a[x];
			a[x]=temp;
		}
	}
	for(i=0;i<=9;i++)
    {
		if(i<9)
         {
			printf("%d>",a[i]);
		}
		if(i==9)
         {
			printf("%d",a[i]);
		}		
	}
	return 0;
}
```
