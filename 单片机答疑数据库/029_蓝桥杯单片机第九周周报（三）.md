### (1)仿真运行操作不当

* **问题描述**:为什么不亮,照着视频做的。
* **问题背景**:0基础第0讲。
* **问题分析**:

  + 学员未启动仿真运行
  + 仿真软件需要手动点击运行按钮才会执行程序
  + 初学者容易忽略仿真软件的操作流程
* **解决方案**:

### (2)数码管小数点配置疑问

* **问题描述**:模版里面这个小数点不全是0吗,不就没有小数点了吗?
* **问题背景**:蓝桥杯大模板代码。
* **问题分析**:

  + 学员对模板中seg_point数组的作用不理解
  + 误认为数组初始化为0就无法显示小数点
  + 不了解数组作为中间变量的设计思路
* **解决方案**:

  + seg_poin数组是用来作中转的,需要加小数点就只需要改数组的值,不用改底层代码方便操作。这是一种解耦设计,通过修改数组元素值(例如seg_poin[2]=1)即可控制对应位置的小数点显示,提高了代码的灵活性和可维护性。

### (4)数码管小数点不显示

* **问题描述**:到了电压显示界面后,数码管第四位不显示数,小数点也不显示,目的是到达电压显示界面正常显示数和小数点。
* **问题背景**:无具体课程背景。
* **问题分析**:

  + 锁存器代码放错位置
  + 锁存器代码被放在了小数点判断的if-else分支内部
  + 导致只有没有小数点的情况下才会更新数码管
  + 程序逻辑顺序错误

* **解决方案**:

  + 锁存器代码要放在小数点判断代码外面。正确的代码结构应该是:先判断是否需要小数点并设置P0值,然后在if-else分支外执行锁存器操作,确保无论是否有小数点都能正常更新数码管显示。

### (5)数码管高位消隐异常

* **问题描述**:在电压计数界面,数码管的第一位有闪0。
* **问题背景**:无具体课程
* **问题分析**:

  + 高位消隐采用while循环的方式实现
  + while循环会阻塞系统运行
  + 阻塞导致数码管扫描更新不及时
  + 出现闪烁现象
* **解决方案**:

  + 使用三目操作符消隐,替代while循环。例如:`ucled[0] = (value >= 1000) ? (value/1000) : 10;`(假设10对应消隐码)。三目操作符不会阻塞系统,能保证数码管扫描的实时性,避免闪烁。

### (6)LED控制逻辑判断错误

* **问题描述**:无论电压变量大不大于电压显示参数,led0都一直亮,目的是让led0在电压变量小于电压显示参数时亮。
* **问题背景**:无具体课程
* **问题分析**:

  + 电压显示参数设置不合理
  + 参数设置成10后不管输入多少都会小于10
  + 导致判断条件永远为真,LED一直点亮
  + 参数范围与实际采集值不匹配
* **解决方案**:

  + 设置合理的电压显示参数。需要根据实际的电压采集范围设置阈值参数,例如:如果电压范围是0-5V,参数应该设置在这个范围内的合理值(如2.5V对应的ADC值),而不是随意设置。
