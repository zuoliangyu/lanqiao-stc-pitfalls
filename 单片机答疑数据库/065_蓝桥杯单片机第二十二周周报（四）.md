### (1)数码管按键跳转后不熄灭(第九届省赛真题)
* **问题描述**:按下S4键跳转到亮度等级显示界面后,松手数码管不熄灭且按键S6失效。
* **问题背景**:小鹅通课程第九届省赛真题,学员在处理按键跳转逻辑时遇到中断处理代码逻辑错误。
* **问题分析**:
* C语言中if...else if结构导致条件判断异常
* 无法进入else分支处理松手逻辑
* 按键状态机设计不合理
* **解决方案**:
* 改用单if语句替代if...else if结构,确保每个条件独立判断
* 重新设计按键状态机,分离按下和松开事件处理
---
### (2)DAC输出电压测量不准与if...else if逻辑问题(十五届第一次省赛题)
* **问题描述**:DAC输出电压测量不准确,且if...else if语句顺序互换导致结果不同。
* **问题背景**:小鹅通课程十五届第一次省赛题,学员在处理数据判断逻辑时遇到条件覆盖问题。
* **问题分析**:
* C语言中if...else if结构存在逻辑覆盖问题
* 当多个条件可能同时满足时,只会执行第一个匹配的分支
* 数据判断逻辑设计不合理
* **解决方案**:
* 改用单if语句进行独立判断,避免条件覆盖
* 确保每个判断条件互斥,或使用独立if语句
```c
// 修改前(错误):
if (voltage > 3.0) {
// 处理高电压
} else if (voltage > 2.0) {
// 当voltage=3.5时,永远不会执行到这里
}
// 修改后(正确):
if (voltage > 3.0) {
// 处理高电压
}
if (voltage > 2.0 && voltage <= 3.0) {
// 明确条件范围
}
```

### (3)滑动变阻器调节无效与EEPROM数据处理
* **问题描述**:滑动变阻器调节无效(始终显示5.00),且EEPROM读取数据为零。
* **问题背景**:学员在处理参数保存和类型处理时遇到变量类型不匹配问题。
* **问题分析**:
* 变量类型不匹配(需要int型处理)
* 存储时未按分辨率乘10处理
* EEPROM读写时数据精度丢失
* **解决方案**:
* 调整变量类型为int型
* 存储时对控制值乘10,读取时除10
* 确保数据精度在存储和读取过程中保持一致
```c
// 修改前(错误):
float control_value = 5.0;
EEPROM_Write(addr, (u8)control_value); // 直接转换导致精度丢失
// 修改后(正确):
int control_value = 50; // 实际值5.0,存储时乘10
EEPROM_Write(addr, (u8)control_value); // 存储50
// 读取时
int read_value = EEPROM_Read(addr);
float display_value = read_value / 10.0; // 显示5.0
```

### (4)DS18B20温度显示异常与平均值计算错误
* **问题描述**:温度显示异常,平均值计算错误(如湿度14时显示14.4),小数显示不正确。
* **问题背景**:学员在处理DS18B20模块数据时遇到数据精度处理和平均值算法问题。
* **问题分析**:
* 小数显示只显示整数部分
* 平均值算法设计不当
* 未正确处理DS18B20的分辨率(0.0625)
* **解决方案**:
* 远程修改代码,正确处理小数显示
* 强调背熟DS18B20分辨率(乘0.0625)
* 推荐使用V4T模板,避免重复造轮子
```c
// 正确处理DS18B20温度数据:
int temp_raw = DS18B20_Read(); // 读取原始数据
float temp = temp_raw * 0.0625; // 乘以分辨率
int temp_display = (int)(temp * 10); // 转换为整数显示(保留一位小数)
// 平均值计算:
int sum = 0;
for (int i = 0; i < count; i++) {
sum += data[i];
}
int avg = sum / count; // 确保使用整数运算
```
