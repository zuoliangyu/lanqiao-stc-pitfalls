### (1)按键按下去没反应

* **问题描述**:按键按下去没有任何反应,无法触发预期功能。
* **问题背景**:无具体课程背景。
* **问题分析**:

  + 函数名拼写错误
  + 调用了不存在的函数名
  + 编译可能没有报错但实际函数未正确链接
* **解决方案**:

  + 读取函数写成了Kep_Read(),函数名错误。需要调用正确的函数名,检查函数名拼写是否正确,确保与定义的函数名完全一致。

### (2)独立按键流水灯卡死

* **问题描述**:独立按键控制流水灯的启停程序,按下按键1开始流水灯,按下按键2暂停流水灯。上电后8个灯亮一下后全部熄灭,按下按键1后8个灯处于卡死状态。
* **问题背景**:零基础入门二。
* **问题分析**:

  + 延时函数内部实现有问题
  + 延时函数中使用了死循环while(xms)
  + 变量xms没有递减,导致程序卡在死循环中无法退出
* **解决方案**:

  + 使用到的延时函数里有死循环while(xms),但xms变量没有递减。需要换成正确递减的延时函数,例如:

```
// 错误示例
void Delay(unsigned int xms)
{
    while(xms)  // xms永远不会变化,死循环
    {
        // 延时代码
    }
}

// 正确示例
void Delay(unsigned int xms)
{
    while(xms--)  // xms递减,最终变为0退出循环
    {
        // 延时代码
    }
}
```

### (3)按键模块没反应

* **问题描述**:整个按键模块好像没反应,无法进行界面切换。
* **问题背景**:十三届省赛。
* **问题分析**:

  + 学员只写了两个界面(界面0和界面1)
  + 在按键判断里写了if(Seg_Disp_Mode==2)Seg_Disp_Mode=0;
  + switch语句中没有case 2的处理,导致当Seg_Disp_Mode=2时显示异常
  + 界面切换逻辑与实际界面数量不匹配
* **解决方案**:

  + 按键部分改成正确的界面切换逻辑。如果只有两个界面(0和1),界面切换应该写成:

```
// 错误示例
if(Seg_Disp_Mode == 2)  // 不存在界面2
    Seg_Disp_Mode = 0;

// 正确示例
if(Seg_Disp_Mode == 1)  // 只有0和1两个界面
    Seg_Disp_Mode = 0;
else
    Seg_Disp_Mode++;
```

### (4)超声波测距数值异常

* **问题描述**:超声波测距数值异常,无法正常测距。
* **问题背景**:十三届省赛。
* **问题分析**:

  + 发送方波函数中连续两次写Tx=1
  + 应该是先Tx=1再Tx=0,形成方波
  + 并且在读取函数中超声波的发送函数也没有调用
  + 没有触发超声波发送,自然无法接收到回波
* **解决方案**:

  + 修改发送方波函数,Tx=1后应该跟着Tx=0,不是Tx=1。并且在超声波读取函数中要调用发送函数,正确的超声波发送流程为:

```
// 发送超声波脉冲
void Send_Wave(void)
{
    Tx = 1;  // 拉高
    Delay_12us;  
    Tx = 0;  // 拉低,形成脉冲
}

// 读取超声波数据
unsigned int Read_Distance(void)
{
    Send_Wave();  // 先发送超声波
    // 等待接收回波并计算距离
    // ...
}
```

### (5)超声波只有个位变化

* **问题描述**:超声波测距时,显示的数值只有个位在变化,十位、百位不变。
* **问题背景**:十三届省赛。
* **问题分析**:

  + 超声波读取函数中的时间变量类型不正确
  + 使用了unsigned char或其他小范围类型
  + 导致数值溢出,只能显示0-255范围内的个位变化
* **解决方案**:

  + 超声波读取函数中的时间变量需要使用int类型或unsigned int类型。确保变量能够存储完整的计数值,避免数据溢出。

```
// 错误示例
unsigned char time;  // 范围0-255,会溢出

// 正确示例
unsigned int time;  // 范围0-65535,足够存储超声波计数值
```

### (6)数码管乱闪

* **问题描述**:板子刚烧录程序数码管显示正常,操作一段时间后,数码管就开始乱闪。
* **问题背景**:无具体课程背景。
* **问题分析**:

  + 数码管刷新周期设置不当
  + 刷新周期过长导致显示不稳定
  + 运行一段时间后,某些变量或状态变化导致刷新异常
* **解决方案**:

  + 将数码管刷新周期改成100ms。
