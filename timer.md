# 定时与计数模块踩坑

---

## 坑 1：频率计数时频繁关开中断会把 1 秒闸门时间拉歪

> 未知题目 · 发现者：左岚

### 现象

用 `NE555` 做频率计数时，测出来的结果和 `4T` 差得比较远，而且现象不稳定: 有时候正常，有时候异常，看起来像玄学漂移。

### 原因

这类频率测量本质上依赖“精确的 1 秒计数窗口”。如果在计数过程中频繁关中断、开中断，就会拖慢定时器轮询或 `1s` 闸门翻转的时刻，最终不是计数器坏了，而是“这一秒”本身被拉长或压短了。

```c
// 错误写法：在计数窗口中途频繁关开中断
EA = 0;
/* 其他处理逻辑 */
EA = 1;
```

### 解决方法

计数窗口内尽量保持中断连续运行，不要为了别的逻辑去打断它。真要做临界区保护，也只在结果快照或状态切换时短暂处理，不要碰闸门计时主链路。

```c

// 每1000ms（1秒）执行一次
if (++Time_1s == 1000)
{
    Time_1s = 0; // 1秒计时器清零
    // 读取定时器0的计数值，即为过去1秒内的脉冲数（频率Hz）
    Freq = (TH0 << 8) | TL0;
    // 清零定时器0，准备下一个周期的计数
    TH0 = TL0 = 0;
}
```

### 举一反三

- 凡是“固定时间窗计数”的题，精度先看时间窗准不准，再看计数值对不对
- 关中断不只会丢事件，还会悄悄改写时间基准
- 遇到“有时正常有时异常”的频率题，先查有没有在测量窗口里做过临时关中断
