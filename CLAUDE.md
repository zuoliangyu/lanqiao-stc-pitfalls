# CLAUDE.md — 蓝桥杯单片机踩坑知识库

本文件供 AI 助手（Claude）在协助整理本目录文档时参考。

---

## 目录用途

汇总蓝桥杯单片机竞赛中遇到的踩坑经验，按硬件/功能模块分文件沉淀。

---

## 文档规范

- **格式范例**：参见 `example.md`，所有文档必须遵循该格式
- **组织方式**：按模块一个文件，同一模块的所有踩坑追加到同一文件末尾
- **文件命名**：模块英文小写，如 `da.md` / `uart.md` / `ultrasonic.md`
- **届次和发现者**：标注在每个坑的引用行，格式为 `> [届次] · 发现者：[姓名]`
- **发现者**：是具体找到这个坑的人，不代表整篇文档的作者
- **图片资源**：放在 `assets/` 目录下，统一管理
- **语言风格**：正文简体中文，代码注释可中英混用

## 模块文件列表

| 文件 | 模块 |
|---|---|
| `led.md` | LED 亮灭、闪烁、PWM 调光 |
| `seg.md` | 数码管显示 |
| `key.md` | 按键检测 |
| `eeprom.md` | EEPROM 读写 |
| `da.md` | DA 输出 |
| `ad.md` | AD 采集 |
| `ultrasonic.md` | 超声波测距 |
| `uart.md` | 串口通信 |

---

## 协作约定

### 用户提供新踩坑时，AI 应当：

1. 判断属于哪个模块，找到对应文件
2. 按 `example.md` 格式追加到该文件末尾（坑编号顺序递增）
3. 更新 `README.md` 中对应模块的坑数统计

### 整理原则：

- 保留原意，不删减技术细节
- 口语化描述改写为简洁规范的书面语
- 代码示例必须包含错误写法和正确写法的对比
- "举一反三"要总结通用规律

### 常用指令：

- "帮我把这段踩坑整理成文档" → 判断模块，追加到对应文件
- "更新一下索引" → 更新 README.md
- "检查一下格式" → 对照 example.md 检查已有文档
