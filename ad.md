# AD 采集模块踩坑

---

## 坑 1：电位器通道先做浮点缩放导致显示跳零

> 第十六届模拟赛 2 · 发现者：谁无年少轻狂时

### 现象

对 `0x03` 电位器 ADC 通道做数据处理时，如果先写成 `temp * 20 / 51.0` 这类浮点运算，数码管会出现多次跳 0 的现象，测评会因此扣分。同题里的光敏电阻通道不太明显，更容易让人误以为是电位器通道本身有问题。

### 原因

8051 上浮点除法开销很大，放在高频刷新的 ADC 数据处理中，会明显拉长一次采样后的运算时间。电位器通道变化更快、显示刷新更频繁，所以更容易把瞬时显示节奏打乱，表现成“偶发跳 0”。

```c
// 错误写法：在高频 ADC 显示路径里直接做浮点除法
unsigned char temp;
temp = Ad_Read(0x03);
Value = temp * 20 / 51.0;
```

### 解决方法

显示用的数据尽量走整数或定点运算，先乘后除，最后再落到显示变量。确实需要小数时，也优先用“放大 10 倍/100 倍”的定点写法，而不是直接上浮点。

```c
// 正确写法：显示值直接用整数缩放
unsigned char temp;
temp = Ad_Read(0x03);
Value = temp * 20 / 51;
```

### 举一反三

- 8051 上的浮点、32 位除法都很贵，别随手塞进高频刷新路径
- 采样、显示、扫描这类“每轮都跑”的逻辑，优先用整数和定点
- 某个通道更容易出问题，很多时候不是通道特殊，而是该通道的处理路径更重

## 坑 2：PWM 开启后不屏蔽中断导致 AD 采样被污染

> 第十三届国赛 · 发现者：萌新也有国一梦

### 现象

程序一旦开启 PWM，并启用每 `100us` 触发一次的 Timer2 中断，`ADDA` 采样值就会乱跳，甚至周期性跳 0。单独看 ADC 代码没问题，但一和 PWM 同时开就炸。

### 原因

`Ad_Read()` 这类访问通常依赖软件时序。Timer2 中断频率高、优先级也高时，会在采样过程中硬插进来，把总线或采样时序切碎，最后读到被污染的数据。

```c
// 错误写法：采样过程中允许高频中断随时打断
temp = Ad_Read(0x43);
```

### 解决方法

在 AD 采样前后临时关总中断，保证一次采样过程连续完成。若题目允许，也可以改成“停止 PWM 后采样，采样完再恢复”。

```c
// 正确写法：把一次采样保护成原子操作
EA = 0;
Ad_Read(0x43);
Ad_Read(0x43);
temp = Ad_Read(0x43);
EA = 1;
```

### 举一反三

- 软件模拟 I2C、SPI、单总线时，最怕高频中断插队
- “单独运行正常，一开 PWM/定时器就乱跳”时，优先怀疑时序被中断打断
- 关中断要只包住最小关键区，别一关就是大段逻辑
